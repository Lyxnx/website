Week 9 - design

  • MVC design pattern
    • Model → view → controller → user
  • Key questions:
    • Model
      • Who is the client
      • What is the purpose of the site
      • Who is the target audience
      • Who is the competition
    • View
      • How will the page be laid out
      • What colours will be used
      • How will the site look and behave on smaller or larger screens
      • Will different font sizes or families be used
    • Controller
      • Provide navigational tools that are consistent across the site
      • Menus are easier to use if they remain the same across all pages
      • Consistency of colours, structures, words, meanings and spacing
  • Can never design a flawless solution but experience affords fewer mistakes
  • Understand the user requirements
  • Understand physical and cognitive abilities
  • Understand what is good enough given the constraints
  • Bad design enables human error

Week 10 - images

  • Images can have srcsets which list the image sources
    • Eg <img srcset=”img1.png 400w, img2.png 600w”>
    • The browser will select the first image if the space it’s in is 400w - this is a hint. With hints, only one image is loaded at first and is not responsive - the page has to be reloaded
  • Img tag can have the sizes attribute which is closely related to srcset. It helps the browser decide which image to use when the browser is in different widths
    • <img src="i/logo.png srcset=" i/elmer/400x400.png 400w, i/elmer/600x400.png 600w, i/elmer/1200x400.png 1200w" sizes=" (max-width: 400px) 100vw, (max-width: 600px) 80vw, (max-width: 1200px) 60vw">

Week 13 - addressing the web

  • The location of a source is a uniform resource locator, or URL
  • Anatomy of a URL: http://admin:secret@port.ac.uk:80/a/path/?x=4&y=3#part_z scheme://username:password@(authority)host:port/path/?query#fragment
  • Common supported types:
    • ftp://
    • http://
    • https://
    • file://
    • mailto:
  • A DNS is a global record of domains to IP addresses
  • Computers run network services on ports
    • Over 65000 available
    • http - port 80
    • https - port 443
    • ssh - port 22
    • ftp - 21
  • Usernames and passwords CAN be specified as part of the URL, but really should not
  • Query string can be passed via URL, allowing server side scripts to interpret them as variables
    • Because HTTP is stateless (the URL is not unique for each user, for example a payment form will be reset if the page is refreshed) they are often used to indicate where we are in a process
      • For example, remembering some unique ID or a page number
  • http://www.example.com/index.html#cake will load the page and look for an element with the ID of cake and scroll to it
  • <a> (anchor) tags are used to link within html by specifying the URL in the href attribute
    • Absolute URLs can be full and MUST contain the scheme (eg http://)
    • Relative URLs can be used to specify a link relative to the current page. For example, <a href=”about.html”> will link to the about.html file in the same folder
    • URLs which omit the scheme are assumed to be relative
      • <a href=”www.bbc.co.uk”> will not go to the BBC website, but will try and go to the file with that exact name in the same directory
    • The target attribute indicates which window the link should load in
      • _blank opens in a new window
      • _self (default) opens in the same window
      • “framename” opens the link in the named frame (evil, don’t do this)
      • Use _blank sparingly - this assumes the user has a system capable of displaying more than one browser window

Week 17 - accessibility

  • Accessible sites improve search rankings
  • Perceivable
    • Provide text alternatives for any non text content
    • Provide alternatives for time based media
    • Create content that can be presented in different ways
    • Make it easier for users to see and hear content
  • Operable
    • Make all functionality available from a keyboard
    • Provide users enough time to read and use content
    • Do not design content in a way that is known to cause seizures (or give a good warning ahead of time)
    • Provide ways to help users navigate, find content, and determine where they are
  • Understandable
    • Make text content readable and understandable
    • Make web pages appear and operate in predictable ways - we know how annoying it is when it works completely opposite as expected
    • Helps users avoid and correct mistakes - assume the user is an idiot and has no clue what they’re doing - prevent them from even making a mistake
  • Robust
    • Maximise compatibility with current and future user agents, including assistive technologies
  • ARIA - accessible rich internet applications
    • Indicates to assistive technologies the behaviour of custom controls
    • Define landmark roles of page areas
    • Set an element to not show on assistive tech with aria-hidden=”true”
